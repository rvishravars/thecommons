# Screenshots Guide

## Required Screenshots

Please capture the following screenshots for the main README:

### 1. Assembly Canvas (`assembly-canvas.png`)
**What to capture:**
- Split-view interface showing Section 1 (Spark Narrative) on the left.
- Sections/Modules picker or active module on the right.
- Show the bottom tab strip for section switching.
- Show the stability indicator in the top header (e.g., 2/8 Stable).

**How to capture:**
- Load an existing enhanced spark.
- Take a full browser screenshot.
- Save as `assembly-canvas.png`.

### 2. Building Blocks (`building-blocks.png`)
**What to capture:**
- Close-up of the right column showing a technical section (e.g., Hypothesis).
- Maturity-aware headers and character counters.
- Maximize/Restore icons in the header.

**How to capture:**
- Focus on the right column content area.
- Save as `building-blocks.png`.

### 3. Full-Screen Editor (`fullscreen-editor.png`)
**What to capture:**
- A section expanded to full-screen mode.
- Show the dark overlay and the close button.
- Focused textarea for high-density drafting.

### 4. Quiz Mode (`quiz-mode.png`)
**What to capture:**
- The Improve Spark / Maturity Audit modal.
- Focus on the provider selection or the reasoning logs screen.

### 5. Spark Selector & Preview (`spark-selector.png`)
**What to capture:**
- Left sidebar showing sparks with the new stability ratios (e.g., 3/8).
- Repository URL input at the top.

---

## Screenshot Specifications
- **Format**: PNG
- **Theme**: Dark Mode
- **Content**: Use actual enhanced spark data for realistic examples.
- **Resolution**: 1080p+ recommended.
